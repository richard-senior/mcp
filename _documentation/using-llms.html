What happened here? (Context is important, again)

Just like your brain neural networks (MLPs etc.) are a collection of things which are interlinked at varying strength.
The strength this linkage is dynamic being based on context. But there is a natural range to this linkage.
For example: The connection between the words 'Flint' and 'Stone' is naturally stronger than the connection between the words 'Flint' and 'Antelope'?

There are around a million words in English. Add to that symbols like ";" and "â˜ ", word fragments, acronyms and things imagined by a computer, and you can get into the tens of millions
If you were to connect everything to everything you would need 10's of trillions of connections.
But there's not much call for 'Flint Antelopes' so not ALL things need to be interconnected in reality.
In fact if an LLM has too many connections it can cause the LLM to 'over fit' or 'hallucinate'.
In fact perhaps the hardest part of creating an LLM is managing this kind of unecessary linkage.

I have a pet theory that the side effect of your brain making pointless connections causes you to laugh

As the LLM operates and the context shifts, the strength of the links (Parameters) between each 'word' fluctuates. The LLM follows the strongest links and outputs what it thinks is the most likely next 'token'.

When you move your eyes to the side as you concentrate, you're doing the same thing
What happened when I asked the LLM to try again without providing extra context? If you can find any difference between that and what we call 'thinking' I'd like to know.

Obviously this is a very simplistic explanation, because the reality is hideously complicated and requires a PHD in Data Science.

Prompts

In practice there is little difference between a rule and a prompt. Both simply provide 'context'.
Both are mechanisms for altering the behaviour of the LLM but their effects are subtly different.

An example prompt can be found here
Rules are about specific details whereas prompts have the effect of putting the LLM into a particular 'mindset'.
This is usually done with a 'role' statement that begins with the words 'You are..'
I aren't going to cover prompts here because I personally do not use them too much, preferring rules in their place.

MCP (Model Context Protocol)

Just as we can expand the knowledge of an LLM with rules, we can expand its reach with MCP.

As the name suggests MCP is a (transport-agnostic.. JsonRPC) 'protocol' originally pioneered by Anthropic but now adopted by almost every LLM.

This protocol allows the LLM to interface effectively run other code or load new informatoin etc.

Below is an example of a 'conversation' between the client (the LLM) and the server (an 'MCP server').

This particular conversation used STDIO as the 'transport' layer, but this could have been done over HTTP or any other transport layer protocol.

Code

First lets just redefine the terms.

All online documention talks about the 'client and the server' in relation to MCP. This is confusing for people used to web developement.
The server is our MCP code and it CAN be a 'server' (code that accepts and returns HTTP over TCPIP) but it doesn't have to be.
In fact it doesn't even have to run permanently, it can be just invoked when needed, perform its task and then exit.

You can use absolutely anything to write an MCP tool because of the way they're integrated using the 'mcp.json' config file which sits in your global amazonq config directory (usually ~/.aws/amazonq/mcp.json) : 


{
   "mcpServers": {
      "mcp": {
         "name": "mcp",
         "command": "/Users/richard/mcp/mcp",
         "description": "A custom Golang MCP server",
         "timeout": 20000
      },
    "digital-io": {
         "name": "digital-io",
         "command": "/Users/richard/mcp/_digital-io/digital-io-server",
         "args": ["-mcp"],
         "description": "A bank of IO which may be used by the LLM",
         "timeout": 20000
      }
   }
}

In the above config file we can see that there are 2 MCP servers defined.
Notice that the definition has a 'command' field, which is literally a bash command (also see the args field)
If whatever that command invokes can accept jsonrpc over STDIO then it can be an MCP server.
To create one then you would write code which handled the transactions listed above.

For example if your weapon of choice is node then first see this:
Which is a Node.JS MCP server implementation.
You would then use an mcp.json entry something like: 


{
  "mcpServers": {
    "mcp-server": {
      "command": "node",
      "args": [
        "/Users/YOUR_USER/mcp-server/build/index.js"
      ]
    }
  }
}

There are many many examples in all manner of languages including Python, Java etc.
I choose GO because it's erm, the best thing for this particular job.

Transport and Engines

Sadly Amazon Q doesn't currently support JsonRPC over HTTP, at least not during my attempts.
However if you wanted to write a communal MCP server (which we absolutely should) then you can use a protocol converter. This is a simple piece of code that converts JsonRPC over STDIO to JsonRPC over HTTP. In fact this is often just a bash script.

Appendix 2. RAG (Retrieval Augmented Generation)

It is somewhat annoying that RAG is an acronym for "Red Amber Green" as it makes searching our wiki rather frustrating

Perhaps the most powerful way of extending the functionality of an LLM is RAG.
RAG is essentially a system by which new knowledge can be implanted natively into the LLM as if the LLM had been initially trained on that data.

For example a particular system or corpus of documented knowledge may be so important to a company that they want the LLM? to 'know' about it. That is, the LLM will not need to 'learn' about it again and again, by parsing context on every session.
This massively reduces hallucination when deal with large amounts of data.

RAG is at the moment a complex process of parsing data in various forms (code, documentation, conversation snippets.. anything) into a form that the LLM can natively understand. This involves the use of various technologies such as 'Vector Datbases' etc.
There are tools that can help such as 'LangChain' and 'Amazon Bedrock', but even so the process is still quite convoluted.

I can't give an example because as yet I haven't actually tried RAG
